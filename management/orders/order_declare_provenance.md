# Order: Declare Provenance

## Direction

The content-quality program proved one lesson above all: the estate's defect
was never invention, it was invention wearing a source's name. The house now
carries a provenance rule in the coding contract: every production declares
its class - sourced, free, or unverified - in its reference warrant, reviews
judge a board only against the class it claims, a marked invention reviews as
clean, and unmarked provenance defaults to sourced so silence never launders
invention. The rule is written for portability to every khai house.

## Orders

1. The Theatre Manager (Edvard Collin) must keep the warrant classes
   truthful: sourced boards anchor their register entry and edition; free
   boards name what they invent; unverified boards name their resolution
   point.
2. The Playwright (Hans Christian Andersen) must, in every future staging or
   restaging, choose and declare the class in the same change that stages
   the content.
3. The Choregos (Nicias and Pericles) must treat class changes as rulings:
   a board moves between sourced, free, and unverified only by order.
4. The Roadie (Vilhelm Pedersen) must adopt the canon-borne key via the kit
   bump and backfill the per-file declarations in the same change: sourced by
   default, free and unverified where the reviews and rulings established
   them.

## Implementation

This governance pull request carries the house-side pointer in `CLAUDE.md`
and this order; it ships no package content. The rule itself lands upstream
(ChBrain/khai, arch/declare-provenance); the house adopts it via the kit
bump with the per-file backfill as a `play/` pull request.

## Targets

- [x] Review semantics are stated: a board is judged against its declared class only
- [x] The kit-bump adoption with per-file backfill is ordered as a following play-lane change
- [x] The rule text is portable to other khai houses
